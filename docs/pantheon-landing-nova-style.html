PANTHEON

Path 1 Engine for PlayStation 2

“The Emotion Engine as conductor. VU1 as geometry worker. DMA/VIF as law.”

v1.0.0-beta.1 — Path 1 baseline

Pantheon is a deterministic, contract-driven Path 1 rendering framework for the PlayStation 2: the EE orchestrates 128-bit DMA/VIF1 command chains; VU1 microcode (shader.vsm) performs transforms and XGKICK to the GS. No high-level wrapper — the same discipline first-party teams used for shipping geometry on fixed silicon.

Authoring stays in Softimage 3D: ASCII .hrc through hrc2ps2.py into hardware-safe *_data.h quadword layouts. The reference build delivers a walkable Path 1 floor grid, Softimage skydome, GTA: San Andreas–inspired timecycle, and a boot sequence tuned for real hardware and PCSX2 — locked 60 FPS baseline on the acceptance matrix.

1. Highlights (current baseline)

  • Strict Path 1 pipeline: EE → VIF1 → VU1 → GIF → GS; shared memory contract in pantheon_path1_contract.h.
  • Softimage → PS2: Floor and skydome meshes as exported headers; checkerboard Path 1 floor with world-anchored tiling and deck snap.
  • Atmosphere: Data-driven sky horizon/zenith, weather slots, smooth temporal blend — baby-blue overcast through purple dusk.
  • Boot: Luma ramp, centered title animation, Path 1 world reveal without clear-channel strobing; libdraw overlay + GS FINISH sync.
  • Toolchain: make -f Makefile.world → floor.elf (PS2SDK / ee-gcc + dvp-as).

3. Operational protocol: build artifact

From a provisioned PS2DEV tree, produce the reference ELF in one pass:

TASK: “Compile Pantheon world demo (Path 1 + hybrid clear).”

EXEC: git clone https://github.com/94BILLY/Pantheon.git && cd Pantheon && make -f Makefile.world clean && make -f Makefile.world

Strict Path 1 profile (no CPU overlay): append EE_CFLAGS='-DPANTHEON_RENDER_PROFILE=1' to the make invocation — see BETA_RELEASE.md on the repo.

4. Installation sequence

Option A (Recommended):

  • Host toolchain: Install PS2SDK / ps2dev so ee-gcc and dvp-as are on PATH (see project README).
  • Run: make -f Makefile.world → flash or run floor.elf via PS2 loader or PCSX2 with a legitimate BIOS.

Option B (Asset authoring):

  1. Export: Softimage → ASCII .hrc
  2. Crunch: python3 hrc2ps2.py <name>.hrc
  3. Integrate: Include generated *_data.h; rebuild floor.elf
