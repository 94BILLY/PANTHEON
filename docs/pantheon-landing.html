PANTHEON

Path 1 Game Engine for the Playstation 2

PlayStation 2 · Path 1 · VU1 · 60 FPS

“The performance and discipline of first-party silicon: the CPU streams work to the hardware; a vector unit prepares geometry and feeds the graphics chip directly.”

v1.0.0-beta — released

Pantheon is a bare-metal Path 1 PlayStation 2 engine: the Emotion Engine issues DMA/VIF1 work; VU1 microcode in shader.vsm transforms vertices, packs GIF primitives, and XGKICKs the Graphics Synthesizer—the same architectural line as late first-party titles, without a thick software renderer in the middle.

This repository is a public technical record and a reproducible whitebox (boot, skydome, floor, orbit camera). It is not a starter kit; there is no LICENSE. Tools hrc2ps2.py and obj2ps2.py emit data in the shape the bus expects—integration is on you. Policy and build details: README and GETTING_STARTED.md on GitHub.

1. What’s included

  • Path 1 pipeline: EE → VIF1 → VU1 → GIF → GS.
  • Asset workflow: Softimage .hrc via hrc2ps2.py; Wavefront .obj via obj2ps2.py.
  • Walkable floor grid and timecycle sky (smoothed palettes).
  • 60 FPS target for this whitebox; boot sequence WWW.94BILLY.COM and orbit camera included.

4. Operational protocol: build artifact

On a host with PS2SDK and ee-gcc, reproduce the reference ELF exactly as in the repo:

EXEC: git clone https://github.com/94BILLY/PANTHEON.git && cd PANTHEON && make -f Makefile.world clean && make -f Makefile.world

5. From mesh to build

Reference wiring lives in floor.c:

  1. Clone and build
    git clone https://github.com/94BILLY/PANTHEON.git && cd PANTHEON && make -f Makefile.world
  2. Create geometry — Blender / DCC: python3 obj2ps2.py your_model.obj your_model_data.c model_vertices
    Softimage: ASCII .hrc → python3 hrc2ps2.py your_model_ascii.hrc
  3. Integrate — wire generated .c / headers like floor_data.h; rebuild.
  4. Run — floor.elf in PCSX2 or on PS2.
