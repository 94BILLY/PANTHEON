PANTHEON

Path 1 Engine for PlayStation 2

PlayStation 2 | Path 1 | VU1 | 60 FPS

Built for the real PS2 pipeline: the CPU streams work to the hardware; a dedicated vector unit prepares geometry for the graphics chip.

v1.0.0-beta — Path 1 baseline

Pantheon is Path 1 on real silicon vocabulary: EE schedules DMA/VIF1; VU1 microcode transforms, packs GIF, XGKICK to GS. Not a tutorial repo. If that sentence is opaque, the manuals come first—then this tree reads as what it is.

Public showcase of the work—not a starter kit, not permission to ship derivatives. No LICENSE; rights reserved. In-repo tools: hrc2ps2.py (Softimage .hrc), obj2ps2.py (Wavefront .obj)—integration with the Path 1 contract is on you.

This baseline is a technical record and reproducible whitebox—not a product pitch, not “make your own PS2 game,” and not permission to treat the tree as a template for unrelated shipping work.

1. What’s included

  • Path 1 pipeline: EE → VIF1 → VU1 → GIF → GS.
  • Asset workflow: Softimage .hrc via hrc2ps2.py; Wavefront .obj via obj2ps2.py.
  • Walkable floor grid and timecycle sky (smooth day/night palettes).
  • 60 FPS target for this whitebox; boot intro and orbit camera included.

EXEC: git clone https://github.com/94BILLY/PANTHEON.git && cd PANTHEON && make -f Makefile.world clean && make -f Makefile.world

5. From mesh to build

A straightforward path to your own geometry (the shipping demo shows the wiring in floor.c):

  1. Clone and build
    git clone https://github.com/94BILLY/PANTHEON.git && cd PANTHEON && make -f Makefile.world
  2. Create geometry
    Modern route: Model in Blender (or similar), export .obj, then
    python3 obj2ps2.py your_model.obj your_model_data.c model_vertices

    Classic route: Softimage 3D → ASCII .hrc →
    python3 hrc2ps2.py your_model_ascii.hrc
  3. Integrate
    Wire generated headers or .c into the project the same way floor.c uses floor_data.h / skydome data; rebuild with make -f Makefile.world.
  4. Run
    Boot floor.elf in PCSX2 or on a PS2.
